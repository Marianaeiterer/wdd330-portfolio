Notes from Doing Stuff with Web Things:

  • Section tag requires the full width of the page, we can use this tag to take advantage and help place the items on the page. I would use this tag before but didn't have this idea, this concept helped me see a useful way to use the section besides just separating and organizing the code.
  • Get users input use the “value” word (input and textareas)
    - Example: var storyName = document.getElementById(“name_input”).value
  • Show something for the user use “innerHTML” (sections)
    - Example: document.getElementById(“story_display”).innerHTML = storyHTML
  • JavaScript's local storage uses what techies call 'key-value pairs.'
  • In the story editor code, for example, the story name is the key and the story is the value
    - localStorage.setItem(storyName, storyHTML)
  • Make statements when working with “if's” helps to decide what to use, and “else if” or just “else”
  • The function parseInt gets the number text and returns it as an integer or returns NaN (Not a Number) if the text is not a number.
  • Box shadow property has 5 things you can send to it.
    - Example: box-shadow: 0px 0px 5px #666666 inset;
    The first 2 values are required the rest is optional
    Description of each part:
    1. The first is how far the shadow is shifted horizontally
    2. The second is how far the shadow is shifted vertically
    3. The third value is how many pixels will be used to blur out the shadow
    4. The fourth value is the shadow color (before it is blurred)
    5. The fifth “inset” means that the shadow will be inside the box (to be outside of the box we don’t put anything)
  • JSON is used to convert anything to text, it converts in a way that the text can be converted back to the original thing.
  • JSON.parse: get the string representing an array and converts it to a JavaScript array
  • JSON.stringify: converts stuff into a string
  • resize: none; -- don't allow the user to resize the textarea overflow: scroll; -- allow scroll the text, for big notes
  • Multiply the text the user enters by the number 1, JavaScript will convert the text to a number

Notes Master Mobile UX:

Design mobile first gives a better experience for the user, it's simpler and more beatuful. It's easier to grow the website and maintain it. Start small and scale up. Mobile first helps concentrate in the content. Mobile is not a fad.

  • Be There. To help move someone along their journey, and deepen their loyalty
  • Be Useful. Provide what the users want
  • Be Quick. A lot of steps makes the user frustrated.
  • Reachability Matters.
  • Speed Matters.

Making things accessible for mobile users reduces frustration. Forms need to contain the necessary information, and be simple and large enough for mobile. Think outside the box when it comes to advertisements is important, the users came to your website for your content, so make it the priority in the page.
