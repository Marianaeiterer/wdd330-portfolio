Notes from Chapter 9: The Window Object:

  • Every environment in JavaScript has a global object. In a browser the global object is the window object.
  • The Browser Object Model (or BOM for short) is a collection of properties and methods that contain information about the browser and computer screen.
  • A way of assigning the variable global to the global object:
    Example:
    // from within the global scope
    const global = this;
  • Global variables can be accessed in all parts of the program. Global variables are actual properties of a global object.
    Example:
    x = 6; // global variable created
    << 6
    window.x // same variable can be accessed as a property of the window object
    << 6
    // both variables are exactly the same
    window.x === x;
    << true
  • Some functions we've already met, such as parseInt() and isNaN(), are actually methods of the global object, which in a browser environment makes them methods of the window object
  • The window object has properties and methods that provide information about the user's browser.
  • The window.location property is an object that contains information about the URL of the current page. It contains a number of properties that provide information about different fragments of the URL.
  • Some properties:
    • href property returns the full URL
      Example:
      window.location.href
      << https://www.sitepoint.com/premium/books/javascript-novice-to-ninja
    • protocol property returns a string of the protocol being used
    • host return a string of the domain of the current URL and port number
    • hotname returns a string of the domain of the current URL
    • pathname returns a string of the path that follows the domain
  • windon.open() method is used to open a new window. The first parameter is the URL, the second is the window title and the third is a list of attributes.
    Example:
    const popup = window.open('https://sitepoint.com','SitePoint','width=400,height=400,resizable=yes');
  • The window.screen object contains information about the screen the browser is displayed on.
  • To find the height and width of the screen in pixels we use:
    window.screen.height
    << 1024
    window.screen.width
    << 1280
  • The availHeight and availWidth can be used to find the height and width of the screen, excluding any operating system menus.
    Example:
    window.screen.availWidth
    << 1280
    window.screen.availHeight
    << 995
  • The screen object is more useful in mobile devices.
  • Each window object contains a document object. This object has properties and methods that deal with the page that has been loaded into the window.
  • Cookies are small files that are saved locally on a user's computer. This is a way of storing information that can be retrieved between requests.
  • A restriction of cookies is that they can only be read by a web page from the same domain that set them. This is to stop sites being able to access information about users, such as other sites they have visited.
  • Creating a Cookie:
    Example:
    document.cookie = 'name=Superman';
    << "name=Superman"
  • To append a new cookie, we just need to assign another cookie. It won't overwrite the last one, it will just append it.
    Example:
    document.cookie = 'hero=true';
    << "hero=true"
    document.cookie='city=Metropolis' ;
    << "city=Metropolis"
  • A cookie's value can be changed by reassigning it to document.cookie using the same name but a different value.
  • document.cookie allow us to see the current contents of a cookie jar
  • The window.setTimeout() method accepts a callback to a function as its first parameter and a number of milliseconds as its second parameter.
  • This method returns an integer which is an ID used to reference to that particular method.
  • It can also cancel the timeout using the window.clearTimeout() method.
    Example:
    window.clearTimeout(5);
    << undefined
  • The window.setInterval() method will repeatedly invoke the callback function after every given number of milliseconds.
    Example:
    function chant(){ console.log('Beetlejuice'); }
    const summon = window.setInterval(chant,1000);
    << 6
  • This will show the message “Beetlejuice” in the console every second (1,000 milliseconds).
  • To stop it, we use: window.clearInterval() method and the variable repeat as an argument
    Example:
    window.clearInterval(summon);
  • The setTimeOut() and setInterval() methods can be used to animate elements on a web page.
  • This was the only way to achieve animation using JavaScript until the window.requestAnimationFrame() method was developed.
  • This method works similar to the window. setInterval() method, but it has a number of improvements.
  • Some improvements are making the most of the browser's built-in graphics-handling capabilities, and not running the animation when the tab is inactive, resulting in a much smoother performance.
  • Example of Animation using setInterval() and requestAnimationFrame() methods: Example
  • We notice that the first square using setInterval() is slower than the second square using the requestAnimationFrame() method.

Notes from Chapter 14: HTML5 APIs:

  • The "data-" attribute is a way of embedding data in a web page using custom attributes that are ignored by the browser
  • The names of these attributes can be decided by the developer, but they must use the following format:
    • Start with "data-".
    • Contain only lowercase letters, numbers, hyphens, dots, colons or underscores.
    • Include an optional string value.
  • Examples:
    data-powers = 'flight superSpeed'
    data-rating = '5'
    data-dropdown
    data-user = 'DAZ'
    data-max-length = '32'
  • Each element has a dataset property that can be used to access any data- attributes it contains.
    Example:
    <div id='hero' data-powers='flight superSpeed'>
    Superman
    </div>
    const superman = document.getElementById('hero');
    const powers = superman.dataset.powers;
    << 'flight superSpeed'
  • The HTML5 specification contains a number of APIs that help to gain access to hardware, such as cameras, batteries, geolocation, and the graphics card.

  • HTML5 Web Storage

  • The Web Storage API provides a key-value store on the client's computer that is similar to using cookies but has fewer restrictions, more storage capacity, and is generally easier to use.
  • The Web Storage API has some crucial differences with cookies:
    • Information stored is not shared with the server on every request.
    • Information is available in multiple windows of the browser (but only if the domain is the same).
    • Storage capacity limit is much larger than the 4KB limit for cookies
    • Any data stored does not automatically expire as it does with cookies. This potentially makes cookies a better choice for something like showing a popup once a day.
  • If a browser supports the Web Storage API, the window object will have a property called localStorage, which is a native object with a number of properties and methods used to store data.
    localStorage.setItem('name', 'Walter White'); // saves a value locally
    localStorage.getItem('name');
    << "Walter White" // get the value stored in the name
  • Rather than using the getItem() and setItem() methods, assignment can be used instead.
    localStorage.name = 'Heisenberg';
    console.log(localStorage.name);
    << "Heisenberg";
    localStorage.removeItem('name'); // removes an entry from local storage
    delete localStorage.name; // we also can remove something using the delete operator
    localStorage.clear(); // completely removes everything stored in local storage

  • Geolocation

  • The Geolocation API is used to obtain the geographical position of the device.
  • This means it can be used to find the user's exact location, then link to nearby places or measure the speed at which the user is moving.

  • Web Workers

  • JavaScript is a single-threaded language, meaning that only one process can run at one time. Web workers allow processes to be run in the background, adding support for concurrency in JavaScript
    Example:
    const worker = new Worker('task.js');
  • This function takes the name of another JavaScript file as an argument.
  • If this file exists, it will be downloaded asynchronously. The worker will only start when the file has finished downloading.
  • If the file doesn't exist, an error is thrown.
  • When a worker has completed its task, it can be stopped using the terminate() method from within the main script or from inside the worker script.
    worker.terminate(); //from the main script
    self.close(); //from the worker script
  • Example of worker being used to calculate the factors of a number: Example
  • Without the worker, if the number was too large the page would freeze and we wouldn't be able to do anything else, like change the background while waiting the answer. Using worker though, allows us to calculate the factors without freeze the page.

  • Websockets

  • Websocket is a new protocol that allows two-way communication with a server - also known as push messaging. This means that a connection is kept open and responses are 'pushed' to the client as soon as they are received.

  • Notifications

  • The Notification API allows you to show messages using the system's notifications. This is usually a popup in the corner of the screen, but it changes depending on the operating system.

  • Multimedia

  • HTML5 introduced the <audio> and <video> tags used to insert audio and video clips into a web page. It also introduced a Media API for controlling the playback of the clips using JavaScript.
    Example of adding a video:
    <video src='http://movie.mp4' controls>
    Your browser does not support the video element.
    </video>
  • The content inside the audio and video tags will only display if the browser does not support them.
  • The controls attribute can be added without any value and will display the browser's native controls, such as play, pause, and volume control.
  • Audio and video elements have a number of properties and methods to control the playback of the clip. Some of them are:
    • play() method will start the clip playing from its current position
    • pause() method will pause the clip at its current position
    • volume property is a number that can be used to set the audio volume
    • muted property is a boolean value that can be used to mute the audio
    • currentTime property is a number value that can be used to jump to another part of the clip
    • playbackRate property is used to fast-forward or rewind the clip by changing its value.
    • loop property is a boolean value that can be set to true to make the clip repeat in a loop
    • duration property can be used to see how long the clip lasts
