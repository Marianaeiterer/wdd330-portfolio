Notes about the "this" keyword:

  • A function that is a property of an object is called its method
    Examples:
    user.sayHi = function() {
    alert("Hello!");
    };
    user.sayHi(); // Hello!

    Another way is:
    // first, declare
    function sayHi() {
    alert("Hello!");
    }
    // then add as a method
    user.sayHi = sayHi;
    user.sayHi(); // Hello!
  • We can use a shorter syntax for the methods
    Examples:
    // these objects do the same
    user = {
    sayHi: function() {
    alert("Hello");
    }
    };

    // method shorthand looks better, right?
    user = {
    sayHi() { // same as "sayHi: function(){...}"
    alert("Hello");
    }
    };
  • Methods allow object to act like object.doSomething()
  • If we need to access something stored in the object we can use the keyword "this"
  • To access the object, a method can use the "this" keyword.
    Examples:
    let user = {
    name: "John",
    age: 30,
    sayHi() {
    // "this" is the "current object"
    alert(this.name);
    }
    };
    user.sayHi(); // John
  • We can also use the name of the object, but if we copy user to another variable and overwrite user, the code won't work. Use "this" prevents those mistakes.
  • “this” can be used in any function, even if it's not a method of an object
  • In JavaScript this is “free”, its value is evaluated at call-time and does not depend on where the method was declared, but rather on what object is “before the dot”.
  • A positive thing about this is that a function can be reused for different objects, but this flexibility creates more possibilities for mistakes.
  • Arrow functions are special: they don't have their “own” this. When this is accessed inside an arrow function, it is taken from outside.
  • Six different ways where this can take on new values:
    1. "this" in global context - Outside of any function, the default for "this" is the window object
    2. "this" in object construction - When you create a new instance of an object with the "new" keyword, "this" refers to the instance.
    3. "this" in an object method - Refers to the object itself
    4. "this" in a simple function - "this" is always set to Window in a simple function.
    5. "this" in an arrow function - "this" in an arrow function is always the same as "this" around it. If we use arrow functions within an object method, the "this" context stays as the object, not Window.
    6. "this" in an event listener - "this" is set to the element that fired the event in an event listener

Notes from Chapter 5: Objects

  • Objects have properties that can be any JavaScript value. If it's a function, it's known as a method.
    Example of an object literal:
    const superman = {
    name: 'Superman',
    'real name': 'Clark Kent',
    height: 75,
    weight: 235,
    hero: true,
    villain: false,
    allies: ['Batman','Supergirl','Superboy'],
    fly() {
    return 'Up, up and away!';
    }
    };
  • All objects can change any time when a program is running, even if it was declared using const.
  • The objects can inherit properties from another object, and all objects have the hasOwnProperty() method.
  • To add a new property to an object we simply assign a value to the new property
    Example:
    superman.city = 'Metropolis';
    << 'Metropolis'
  • To change the value of an object's property we use assign.
    Example:
    superman['real name'] = 'Kal-El';
    << 'Kal-El'
  • Any property can be removed from an object, we use the delete operator
    Example:
    delete superman.fly
    << true
  • Nested Objects are objects that contain other objects.
  • Objects are copied by reference, if we create an object that is a copy of another object, both will point to the same object in memory, so if we change the property in one, it will automatically change in the other, because they refer to the same object.

  • Built-in Objects:

  • JavaScript has a number of built-in projects that can be accessed in any part of the program. They provide useful properties and methods.
  • JSON: is a string representation of the object literal notation that we have just seen.
  • JSON methods:
    • The parse() method transforms a JSON string of data into a JavaScript object
    • The stringify() method transforms a JavaScript object into a JSON string data
    • The methods inside objects are ignored by the stringify()
  • The Math Object is an object that has several properties that represent mathematical constants and methods of mathematical operations
  • Date Object contains information about dates and times
  • RegExp Object a regular expression is a pattern that can be used to search strings that match the pattern.

Notes from Chapter 6: The Document Object Model

  • The Document Object Model (DOM): represents an HTML document as a network of connected nodes.
  • We can use JavaScript to access and modify parts of a web page using the built-in object called document.
  • The DOM provides several methods that allow us to access any element on a page. These methods will return an object or a note list.
  • Shortcut Methods:
    • Document.body - returns the body element of a web page
    • Document.images - returns a node list of all the images
    • Document.links - returns a node list of all the <a> elements and <area> elements with an href attribute
    • Document.anchors - returns a node list of all the <a> elements with a name attribute
    • Document.forms- returns a node list of all the forms.
  • Node lists are array-like objects but are not arrays. They can be transformed into an array using Array.from() method.
  • getAttribute() - returns the value of the attribute provided as an argument.
  • setAttribute() - can change the value of an element's attributes. It takes two arguments: the attribute that you wish to change and the new value of that attribute. If an element does not have an attribute, it can add it to the element.
    Example:
    We want to change the class of the element in the wonderWomen variable to 'villain', we could do so using this code:
    wonderWoman.setAttribute('class', 'villain');
    << undefined
    wonderWoman.getAttribute('class');
    << "villain"
  • The remove method will remove a specific class from an element.
  • The toggle method will add a class if an element doesn't have the class and remove the class if the element already has it.
  • The contains method will check to see if the element has a particular class.
  • The innerHTML element property returns all the child elements of an element as a string of HTML. If an element contains lots of other elements, all the raw HTML is returned. It can also be used to place a chunk of HTML inside an element. Also, if we want to insert a large amount of HTML into the document, we can use it to insert the raw HTML as a string instead of creating each element.

Notes from Chapter 7: Events

  • Every time the user interacts with a webpage an event occurs and the program can respond to.
  • Event listeners let the program know that the event happened then the program can respond to it.
    Example of event listener for when the user clicks anywhere in the page:
    document.body.addEventListener("click", doSomething);
  • The addEventListener() method is the recommended way to deal with events
    Example:
    document.body.addEventListener('click',doSomething);
    The first parameter is the type of event
    The second parameter is a callback function that is invoked when the event occurs.
  • Some types of events:
    • Click
    • Mousedown
    • Mouseup
    • Dblclick, double click
    • Mouseover - occurs when the mouse pointer is placed over the element
    • Mouseout - occurs when the mouser point moves away from an element.
    • Mousemove - occurs whenever the mouse moves, while the cursor is placed in the element applied.
    • Keydown - occurs when a key is pressed and continue if the key is held down
    • Keypress - after the keydown and before the keyup. Only occurs for keys that produce character input. This means that it's the most reliable way to find out the character that was pressed on the keyboard.
    • Keyup - occurs when a key is released.
  • removeEventListener() method can remover an event listener
  • preventDefault() method prevents the default behavior. Be careful when using it.
  • Event Propagation is the order that the events fire on each element. When you click on an element, you are actually clicking on all the elements nested inside of it.
  • Bubbling: The event fires on the element clicked first, and then bubbles up firing an event on each parent element until the root.
  • Capturing: fires an event on the root element first and then propagates downwards, firing an event on each child element until it reaches the target element that was clicked on. It happen adding the third parameter to the addEventListener() method.
    Example:
    ulElement.addEventListener('click', (event) =>
    console.log('Clicked on ul'),true);
    liElement.addEventListener('click', (event) =>
    console.log('Clicked on li'),true);
  • Event Delegation: used to attach an event listener to a parent element in order to capture events that are triggered by its child elements.
    Example:
    <ul id='list'>
    <li>one</li>
    <li>two</li>
    <li>three</li>
    </ul>
    ulElement.addEventListener('click',highlight);
    Now clicking on any list item will highlight that list item as if it was the target of the click event.
